## Task 1: Blog Post Bugs (Fix First)

These are bugs in the current live site that need fixing before any redesign work.

### 1A. Cover Images Are Cropped to Square

**What's wrong:** Blog post cover images render as a square crop instead of the intended 16:9 aspect ratio. The image feels cropped and compressed.

**Where to look:**
- `app/blog/[slug]/page.tsx` — the cover image section uses `aspect-[16/9]` but something may be overriding it
- `components/BlogCard.tsx` — the card thumbnail might also have the wrong aspect ratio
- Check any global CSS in `app/globals.css` that could affect `.prose img` or override aspect ratios

**Success criterion:** Cover images display at 16:9 on the blog post page. Blog card thumbnails also display at the correct ratio on the blog list page.

### 1B. Table of Contents Links Don't Navigate

**What's wrong:** The sticky Table of Contents on the right side of blog posts (`components/TableOfContents.tsx`) renders correctly, but clicking an item does not scroll to the corresponding heading in the article.

**Where to look:**
- `components/TableOfContents.tsx` — generates `<a href="#heading-id">` links
- `lib/utils.ts` → `extractHeadings()` — generates heading IDs server-side from the raw MDX string
- `lib/content.ts` — the MDX processing pipeline. Check if `rehype-slug` is installed and configured. `rehype-slug` adds `id` attributes to rendered `<h2>` and `<h3>` elements. If it's missing, the rendered HTML has no IDs for the ToC links to target
- The IDs generated by `extractHeadings()` must exactly match the IDs that `rehype-slug` produces in the rendered HTML

**Root cause hypothesis:** Either `rehype-slug` is not in the MDX pipeline, or the ID generation logic in `extractHeadings()` differs from what `rehype-slug` produces (e.g., different handling of special characters, backticks, or formatting).

**Success criterion:** Clicking any ToC entry smoothly scrolls to the correct heading in the blog post.

### 1C. No Code Blocks Showing in Blog Posts

**What's wrong:** Blog posts that should contain code snippets and demo outputs show no code at all. The content may be missing from the generated MDX, or the rendering pipeline may be stripping it.

**Where to look:**
- Check the actual `.mdx` files in `content/blog/` — do they contain fenced code blocks (` ``` `)? If not, the generation pipeline is the problem
- `scripts/generate_post.py` — the blog generation script. It should be instructed to include code examples and demo outputs
- `lib/content.ts` — check that `rehype-pretty-code` (Shiki) is configured in the MDX processing pipeline
- `app/globals.css` — has styles for `.prose pre` and `.prose code` that should render code blocks properly

**Success criterion:** Blog posts contain syntax-highlighted code blocks that illustrate the concepts discussed. Code should be real, runnable examples — not pseudocode.

### 1D. Blog Post Figures/Plots Quality

**What's wrong:** The generated blog posts should include high-quality data visualizations (plots, figures, charts) with clear legends, readable labels, and visually appealing styling. Currently they are either missing or low quality.

**Where to look:**
- `scripts/plot_style.py` — defines the matplotlib style used for all generated figures
- `scripts/run_demo.py` — generates demo output figures for blog posts
- `scripts/generate_inline_images.py` — generates inline illustrations

**Success criterion:** Figures are publication-quality: clear axis labels, legible legends, consistent color palette (use the editorial palette from `lib/design-tokens.ts`), appropriate sizing, and proper captions in the blog post.
